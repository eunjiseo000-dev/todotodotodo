# PRD (Product Requirements Document)

## ToDoToDoToDo

---

## 요약

**한 줄 정의**
사용자가 할일을 쉽게 관리할 수 있는 웹 기반 할일 관리 애플리케이션

**목표**

- 사용자별 개인화된 할일 관리 제공
- 할일의 상태(진행/완료)와 기간(시작일/종료일) 체계적 관리
- 실수 방지를 위한 휴지통 및 복구 기능 제공

---

## 주요 기능

| 기능            | 설명                                                     |
| --------------- | -------------------------------------------------------- |
| 회원가입/로그인 | 이메일, 비밀번호로 개인 계정 관리                        |
| 할일 추가       | 제목, 시작일, 종료일로 할일 생성                         |
| 할일 조회       | 진행중/완료/휴지통으로 분류된 할일 목록 확인             |
| 할일 수정       | 기존 할일의 내용, 날짜 변경                              |
| 할일 완료 처리  | 진행중 할일을 완료 상태로 변경 (자동으로 완료 목록 이동) |
| 할일 삭제       | 할일을 휴지통으로 이동 (소프트 삭제)                     |
| 할일 복구       | 휴지통에서 활성 할일로 복원                              |
| 할일 영구삭제   | 휴지통의 할일 완전 제거 (복구 불가)                      |
| 우선순위 변경   | 할일의 표시 순서 조정 (드래그 앤 드롭)                   |

---

## 사용자

**Primary User**: 할일을 체계적으로 관리하고 싶은 개인 및 소규모 팀 사용자

**사용자 니즈**:

- 빠르게 할일을 추가하고 싶음
- 마감일을 한눈에 파악하고 싶음
- 실수로 삭제한 할일을 복구하고 싶음
- 우선순위에 따라 할일 순서를 조정하고 싶음

---

## 화면 구조 (2개)

### 화면 1: 할일 관리 대시보드

- 3개 탭: 진행중 / 완료 / 휴지통
- 각 탭에서 할일 목록 표시 (제목, 시작일, 종료일)
- 할일별 액션: 완료(체크), 수정, 삭제, 우선순위 변경 버튼
- 새 할일 추가 버튼

### 화면 2: 할일 추가/수정 폼

- 제목 입력란 (필수)
- 시작일 선택 (필수)
- 종료일 선택 (필수, 시작일 이후)
- 저장/취소 버튼

## 업무 규칙

| 규칙               | 내용                                              |
| ------------------ | ------------------------------------------------- |
| **날짜 검증**      | 시작일 ≤ 종료일 (위반 시 에러 메시지)             |
| **삭제 정책**      | 삭제 시 소프트 삭제 적용, 휴지통에서 복구 가능    |
| **접근 권한**      | 사용자는 자신의 할일만 접근 가능                  |
| **완료 자동 분류** | 완료 체크 시 자동으로 완료 목록으로 이동          |
| **영구삭제**       | 휴지통에서 영구삭제 시 데이터베이스에서 물리 삭제 |

## 성공 기준

### 출시 조건

- ✓ 모든 기능(회원가입, 로그인, 할일 관리) 정상 작동
- ✓ 사용자 데이터 bcrypt로 암호화 저장
- ✓ 로그인 기반 접근 제어 구현
- ✓ 모바일/태블릿에서 기본 기능 사용 가능

### 성공 지표 (출시 후 3개월)

- **월간 활성 사용자(MAU)**: 300명 이상
- **평균 응답 시간**: 1초 이내
- **사용자 만족도(NPS)**: 30 이상

### 실패 판단

- MAU가 100명 이하로 정체
- 중대 보안 문제 발생
- 응답 시간이 지속적으로 3초 이상

| --------------- | ----- | ---------------------------- |
| 설계            | 0.5일 | DB 설계, 화면 레이아웃 설계  |
| 백엔드 개발     | 1일   | 인증, 할일 CRUD, 휴지통 기능 |
| 프론트엔드 개발 | 1일   | 대시보드, 폼 화면 구현       |
| 테스트          | 0.2일 | 기능 테스트 및 버그 수정     |
| 배포            | 0.3일 | 배포 및 모니터링 설정        |

## 기술 스택

| 계층         | 기술                | 설명                           |
| ------------ | ------------------- | ------------------------------ |
| 백엔드       | Node.js/Express     | REST API 개발                  |
| ORM          | Prisma              | PostgreSQL 데이터 접근 계층    |
| 데이터베이스 | Supabase PostgreSQL | 사용자 및 할일 데이터 관리     |
| 프론트엔드   | React 18.x          | 웹 UI 구현                     |
| 호스팅       | Vercel              | 프론트엔드 + 백엔드 API 호스팅 |
| 보안         | bcrypt              | 비밀번호 해시 저장             |
|              | JWT                 | 토큰 기반 인증 (24시간 유효)   |

## 제약사항 및 위험

### Out of Scope (미포함)

- 팀 협업 기능
- 반복 작업 자동화
- 외부 캘린더 연동
- 파일 첨부
- 알림/리마인더

### 기술 제약

- 클라우드: Vercel (프론트엔드/백엔드 통합 호스팅)
- 모바일: 반응형 웹으로 지원 (네이티브 앱 미포함)

### 주요 위험

| 위험               | 영향 | 대응                                     |
| ------------------ | ---- | ---------------------------------------- |
| 개발 지연          | 높음 | 1주 단위 진행상황 검토, 필요시 기능 축소 |
| 보안 취약점        | 높음 | 보안 코드 리뷰, 주기적 보안 테스트       |
| 사용자 확보 어려움 | 중간 | SNS 마케팅, 초기 사용자 인센티브         |
