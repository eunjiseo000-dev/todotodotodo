# ToDoToDoToDo 와이어프레임 설계

## 개요

본 문서는 ToDoToDoToDo 프로젝트의 모든 화면에 대한 상세 와이어프레임과 사용자 인터페이스 설계를 포함합니다.

**참고 문서:**
- [3-prd.md](./3-prd.md) - 기능 요구사항
- [5-project-structure.md](./5-project-structure.md) - 설계 원칙
- [4-user-scenarios.md](./4-user-scenarios.md) - 사용자 시나리오

**디자인 원칙:**
- 단순하고 직관적인 UI (오버엔지니어링 금지)
- 사용성 중심의 설계
- 시각적 계층 구조 명확화
- 일관된 컴포넌트 사용
- 모바일 우선 반응형 디자인

### 주요 컴포넌트

| 컴포넌트 | 타입 | 속성 | 설명 |
|---------|------|------|------|
| 로고/타이틀 | Text | 대형, 굵게 | 브랜드 아이덴티티 |
| 이메일 입력 | Input | type="email", required | RFC 5322 형식 검증 |
| 비밀번호 입력 | Input | type="password", required | 마스킹, 8자 이상 |
| 비밀번호 확인 | Input | type="password", required | 비밀번호 일치 검증 |
| 이름 입력 | Input | type="text", required | 2-50자 |
| 도움말 텍스트 | Text | 작게, 회색 | 입력 규칙 안내 |
| 회원가입 버튼 | Button | primary, full-width | 제출 액션 |
| 로그인 링크 | Link | 텍스트 링크 | /login 페이지로 이동 |

### 사용자 상호작용

1. **입력 필드 포커스**
   - 포커스 시 테두리 색상 변경 (파란색)
   - 포커스 아웃 시 실시간 검증

2. **입력 검증**
   - 이메일: 형식 검증 (예: `test@example.com`)
   - 비밀번호: 8자 이상, 영문+숫자+특수문자 조합
   - 비밀번호 확인: 비밀번호와 일치 여부
   - 이름: 2-50자 범위

3. **에러 표시**
   - 각 필드 하단에 빨간색 에러 메시지 표시
   - 예: "이메일 형식이 올바르지 않습니다"
   - 예: "비밀번호가 일치하지 않습니다"

4. **회원가입 버튼 클릭**
   - 모든 검증 통과 시 API 요청 (POST /api/auth/signup)
   - 로딩 상태: 버튼 비활성화 + 스피너 표시
   - 성공: 로그인 페이지로 리다이렉트
   - 실패: 에러 메시지 토스트 표시

5. **로그인 링크 클릭**
   - 로그인 페이지(/login)로 이동

### 반응형 고려사항

- **Desktop (768px 이상)**
  - 중앙 정렬 카드 레이아웃 (최대 너비 400px)
  - 여백 충분히 확보
  - 입력 필드 높이 48px

- **Mobile (767px 이하)**
  - 전체 화면 활용 (좌우 패딩 16px)
  - 입력 필드 높이 44px (터치 최적화)
  - 버튼 높이 48px (손가락 터치 영역)
  - 세로 스크롤 허용

### 접근성 (Accessibility)

- **키보드 네비게이션**
  - Tab 키로 필드 간 이동
  - Enter 키로 폼 제출

- **스크린 리더**
  - 각 입력 필드에 `label` 요소 연결
  - 에러 메시지에 `aria-live="polite"` 속성
  - 버튼에 `aria-label="회원가입"` 속성

- **시각적 피드백**
  - 포커스 시 명확한 테두리
  - 에러 시 빨간색 아이콘 + 메시지

### 상태 다이어그램

```
[초기 상태]
   ↓
[입력 중] → [검증 실패] → [에러 표시]
   ↓
[검증 통과]
   ↓
[회원가입 버튼 클릭] → [로딩 상태]
   ↓
[API 요청]
   ↓
[성공] → [로그인 페이지로 이동]
[실패] → [에러 토스트 표시]
```

1. **탭 전환**
   - 탭 클릭 시 해당 카테고리의 할일 목록 표시
   - 진행중: `isCompleted=false, isDeleted=false`
   - 완료: `isCompleted=true, isDeleted=false`
   - 휴지통: `isDeleted=true`
   - 탭 개수 실시간 업데이트

2. **새 할일 추가 버튼 클릭**
   - 할일 추가 폼 모달/페이지 열기
   - 폼 제출 시 목록 새로고침

3. **완료 체크박스 토글**
   - 체크 시: `isCompleted=true` (완료 탭으로 이동)
   - 체크 해제 시: `isCompleted=false` (진행중 탭으로 이동)
   - API 요청: PATCH /api/todos/:id/complete
   - 낙관적 업데이트 (즉시 UI 반영)

4. **수정 버튼 클릭**
   - 할일 수정 폼 모달/페이지 열기
   - 기존 데이터 자동 입력
   - 저장 시 목록 새로고침

5. **삭제 버튼 클릭**
   - 확인 다이얼로그 표시: "휴지통으로 이동하시겠습니까?"
   - 확인 시: API 요청 (DELETE /api/todos/:id) - Soft Delete
   - 목록에서 제거 (휴지통 탭 개수 +1)
   - 성공 토스트: "할일이 휴지통으로 이동되었습니다"

6. **복구 버튼 클릭** (휴지통 탭에서만 표시)
   - API 요청: POST /api/todos/:id/restore
   - 진행중 탭으로 이동
   - 성공 토스트: "할일이 복구되었습니다"

7. **영구삭제 버튼 클릭** (휴지통 탭에서만 표시)
   - 경고 다이얼로그: "영구적으로 삭제됩니다. 복구할 수 없습니다. 계속하시겠습니까?"
   - 확인 시: API 요청 (DELETE /api/todos/:id/permanent)
   - 목록에서 완전 제거
   - 성공 토스트: "할일이 영구 삭제되었습니다"

8. **드래그 앤 드롭** (우선순위 변경)
   - 드래그 핸들(⋮⋮)을 잡고 드래그
   - 드롭 위치에 따라 순서 변경
   - API 요청: PATCH /api/todos/:id/reorder
   - 트랜잭션으로 순서 재계산

9. **로그아웃 버튼 클릭**
   - localStorage에서 토큰 삭제
   - AuthContext 상태 초기화
   - 로그인 페이지(/)로 리다이렉트

### 탭별 액션 버튼 표시

| 탭 | 완료 체크박스 | 수정 버튼 | 삭제 버튼 | 복구 버튼 | 영구삭제 버튼 |
|----|--------------|----------|----------|----------|-------------|
| 진행중 | ✓ (비체크) | ✓ | ✓ | ✗ | ✗ |
| 완료 | ✓ (체크됨) | ✓ | ✓ | ✗ | ✗ |
| 휴지통 | ✗ | ✗ | ✗ | ✓ | ✓ |

### 반응형 고려사항

- **Desktop (1024px 이상)**
  - 최대 너비 1200px, 중앙 정렬
  - 할일 아이템: 카드 형태, 그림자 효과
  - 액션 버튼: 아이콘 + 텍스트
  - 날짜: 전체 형식 (2025-11-26)

- **Tablet (768px ~ 1023px)**
  - 전체 너비 활용 (좌우 패딩 24px)
  - 할일 아이템: 간격 16px
  - 액션 버튼: 아이콘만

- **Mobile (767px 이하)**
  - 전체 화면 활용 (좌우 패딩 16px)
  - 탭: 아이콘 + 짧은 텍스트
  - 할일 아이템: 세로 레이아웃
  - 날짜: 짧은 형식 (11-26)
  - 액션 버튼: 크기 44px (터치 최적화)
  - 드래그 핸들: 크기 증가

### 접근성

- **키보드 네비게이션**
  - Tab 키로 요소 간 이동
  - Enter/Space 키로 버튼 클릭
  - 드래그: 키보드로도 순서 변경 가능 (라이브러리 지원)

- **스크린 리더**

  - 할일 아이템: `role="article"`
  - 액션 버튼: `aria-label="할일 수정"` 등
  - 개수 표시: `aria-live="polite"`

- **시각적 피드백**
  - 드래그 중: 투명도 50%, 그림자 증가
  - 호버: 배경색 변경
  - 포커스: 명확한 테두리

   - 모달 닫기

6. **닫기 버튼(X) 클릭 또는 오버레이 클릭**
   - 입력 중이면 확인 다이얼로그: "작성 중인 내용이 있습니다. 취소하시겠습니까?"
   - 확인 시 모달 닫기

### 주요 컴포넌트

| 컴포넌트 | 타입 | 속성 | 설명 |
|---------|------|------|------|
| 제목 | Text | 대형, 굵게 | 할일 제목 |
| 기간 정보 | Text | 아이콘 + 텍스트 | 시작일 ~ 종료일 |
| 생성일 | Text | 아이콘 + 텍스트 | ISO 8601 형식 |
| 수정일 | Text | 아이콘 + 텍스트 | ISO 8601 형식 |
| 상태 | Badge | 색상 구분 | 진행중/완료/휴지통 |
| 액션 버튼 | Button | primary/danger | 완료/수정/삭제 |

### 사용자 상호작용

1. **할일 아이템 클릭 (대시보드에서)**
   - 상세보기 모달 열기
   - 할일 정보 표시

2. **액션 버튼**
   - 완료/수정/삭제: 대시보드와 동일한 동작
   - 액션 후 모달 자동 닫기

### 반응형 고려사항

- **Desktop**
  - 모달 너비 600px
  - 정보 2열 레이아웃 (좌측: 기간, 우측: 생성일/수정일)

- **Mobile**
  - 전체 화면 모달
  - 정보 1열 레이아웃 (세로 정렬)

### 접근성

- **키보드 네비게이션**
  - Tab 키로 버튼 간 이동
  - Esc 키로 모달 닫기

- **스크린 리더**
  - `role="dialog"`
  - 각 정보 항목에 명확한 레이블

### Mobile 최적화 요소

1. **터치 영역**
   - 버튼 최소 높이: 44px
   - 터치 간격: 8px
   - 드래그 핸들: 48px

2. **폰트 크기**
   - 제목: 18px
   - 본문: 16px

4. **모달/폼**
   - 전체 화면 모달 또는 하단 시트
   - 키보드 팝업 시 스크롤 가능

## 11. 애니메이션 및 전환 효과

### 페이지 전환

- **페이드 인**: 0.3초, ease-in-out
- **슬라이드 인**: 0.3초, ease-out (모바일)

### 모달/다이얼로그

- **열기**: 페이드 인 + 스케일 업 (0.9 → 1.0), 0.2초
- **닫기**: 페이드 아웃 + 스케일 다운 (1.0 → 0.9), 0.2초

### 리스트 아이템

- **추가**: 슬라이드 다운 + 페이드 인, 0.3초
- **삭제**: 슬라이드 업 + 페이드 아웃, 0.3초
- **드래그**: 투명도 50%, 그림자 증가

### 버튼 상호작용

- **호버**: 배경색 변경, 0.15초
- **클릭**: 스케일 다운 (0.95), 0.1초

### 토스트 메시지

- **나타남**: 슬라이드 다운 + 페이드 인, 0.3초
- **사라짐**: 슬라이드 업 + 페이드 아웃, 0.3초
