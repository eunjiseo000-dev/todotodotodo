# PRD(Product Requirements Document) 템플릿
## ToDoToDoToDo 프로젝트용

---

## 요약

### 한 줄 정의
[사용자가 쉽게 할일을 관리할 수 있는 웹 애플리케이션]

### 문제
- 할일을 여러 곳에 분산하여 관리 중
- 할일의 진행 상태와 마감일 파악이 어려움

### 해결 방법
- 사용자 로그인 기반 개인화된 할일 관리
- 상태(진행/완료)와 기간(시작일/종료일) 체계적 관리
- 휴지통에서 실수로 삭제한 할일 복구 가능

---

## 기능

| 기능 | 설명 |
|------|------|
| 회원가입/로그인 | 이메일과 비밀번호로 계정 생성 및 로그인 |
| 할일 추가 | 제목, 시작일, 종료일로 새 할일 생성 |
| 할일 목록 조회 | 진행중/완료/휴지통으로 분류된 할일 목록 확인 |
| 할일 수정 | 기존 할일의 정보 변경 |
| 할일 삭제 | 할일을 휴지통으로 이동 |
| 할일 완료 처리 | 진행중인 할일을 완료 상태로 변경 |
| 할일 복구 | 휴지통에서 할일 복원 |
| 할일 영구삭제 | 휴지통의 할일 완전 제거 |

---

## 사용자

**주요 사용자**: 할일을 체계적으로 관리하고 싶은 개인 및 소규모팀

**주요 니즈**:
- 빠르게 할일 추가 가능
- 마감일이 한눈에 보임
- 실수로 삭제한 할일 복구 가능

---

## 화면 구조 (2개)

**화면 1: 할일 관리**
- 진행중 할일 목록
- 완료, 수정, 삭제 버튼
- 탭: 진행중 / 완료 / 휴지통

**화면 2: 할일 추가/수정**
- 제목 입력란
- 시작일 선택
- 종료일 선택
- 저장/취소 버튼

---

## 성공 기준

### 출시 조건
- 모든 기능이 정상 작동
- 사용자 데이터 암호화 처리
- 로그인 기능 정상 작동

### 목표 (출시 후 3개월)
- 월간 활성 사용자 300명 이상
- 평균 응답 시간 1초 이내
- 사용자 만족도 30 이상 (NPS)

## 일정 (MVP: 12주)

| 단계 | 기간 | 내용 |
|------|------|------|
| 설계 | 1주 | 기능 정의, 화면 설계 |
| 개발 | 8주 | 백엔드 및 프론트엔드 구현 |
| 테스트 | 2주 | 기능 테스트 및 버그 수정 |
| 배포 | 1주 | 배포 및 모니터링 |

---

## 기술 스택

| 계층 | 기술 | 설명 |
|------|------|------|
| 백엔드 | Python/Django | REST API 개발 |
| 데이터베이스 | PostgreSQL | 사용자 및 할일 데이터 관리 |
| 프론트엔드 | React | 웹 UI 구현 |
| 인프라 | AWS (EC2, RDS) | 애플리케이션 및 데이터베이스 호스팅 |
| 보안 | bcrypt | 비밀번호 암호화 |

## 주의사항

**포함되지 않는 기능** (향후 추가 예정):
- 팀 협업
- 반복 작업
- 캘린더 연동

**기술 제약**:
- 클라우드: AWS
- 데이터베이스: PostgreSQL
- 모바일: 반응형 웹 지원
