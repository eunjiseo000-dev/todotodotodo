# 사용자 시나리오
## ToDoToDoToDo 프로젝트

---

## 개요

이 문서는 사용자가 ToDoToDoToDo를 실제로 어떻게 사용하는지 보여주는 시나리오들을 정리한 것입니다.

도메인 정의서의 기능들이 실무에서 어떻게 활용되는지, 하루 중 언제 어떤 액션을 취하는지 이해할 수 있습니다.

---

## 1. 전체 사용 흐름

### 시작: 신규 사용자 가입
```
1. 회원가입 페이지 접근
2. 이메일, 비밀번호, 이름 입력
3. 로그인 페이지로 이동
4. 이메일과 비밀번호로 로그인
```

### 일반적인 하루의 흐름
```
아침 (08:00)
→ 로그인 후 할일 목록 확인
→ 오늘의 새로운 할일 2-3개 추가

점심 (12:00)
→ 진행 중인 할일 상태 확인
→ 완료된 할일 체크 (휴지통으로 이동)
→ 완료 목록에서 확인

저녁 (18:00)
→ 남은 할일 확인
→ 내일 할일 추가
→ 일부 할일 우선순위 조정

주간 정리
→ 휴지통의 오래된 할일 영구 삭제
```

## 2. 주요 사용 시나리오

### Scenario 1: 아침 회의 시 할일 추가

**상황**: 팀 회의 중에 새로운 작업이 3개 생김

**액션**:
1. 핸드폰에서 ToDoToDoToDo 앱 열기
2. "할일 추가" 버튼 클릭
3. 각 항목마다 제목, 시작일(오늘), 종료일(3일 후) 입력
4. 저장

**결과**: 3개의 할일이 진행중 목록에 추가됨 (기능: FR-005)

---

### Scenario 2: 점심시간 진행률 확인

**상황**: 점심시간에 오전에 한 일들의 진행 상황을 확인하고 싶음

**액션**:
1. 로그인하여 할일 목록 열기
2. 진행중 탭에서 각 할일의 상태 확인
3. 완료한 할일의 체크박스 클릭

**결과**: 체크된 할일이 완료 목록으로 자동 이동 (기능: FR-007, FR-009)

---

### Scenario 3: 할일 내용 및 일정 변경

**상황**: 오늘 마무리하기로 했던 할일의 마감일이 내일로 연장됨

**액션**:
1. 진행중 목록에서 해당 할일 클릭
2. 종료일을 내일로 변경
3. 저장

**결과**: 할일의 종료일이 수정됨 (기능: FR-006)

---

### Scenario 4: 실수로 삭제한 할일 복구

**상황**: 잘못해서 중요한 할일을 삭제 버튼 클릭함

**액션**:
1. 휴지통 탭으로 이동
2. 삭제된 할일 목록에서 복원할 항목 찾기
3. 복원 버튼 클릭

**결과**: 할일이 진행중 목록으로 다시 돌아옴 (기능: FR-008)

---

### Scenario 5: 마감일 임박 할일 우선순위 변경

**상황**: 종료일이 오늘인 할일이 3개 있는데, 순서를 정해야 함

**액션**:
1. 진행중 목록에서 첫 번째 할일을 마우스로 드래그
2. 가장 중요한 순서로 배열
3. 순서 변경이 자동으로 저장됨

**결과**: 할일들이 새로운 우선순위 순서로 정렬됨 (기능: FR-010)

---

### Scenario 6: 주간 정리 및 영구삭제

**상황**: 금요일 저녁, 지난 주에 휴지통에 넣은 할일들이 더 이상 필요 없음

**액션**:
1. 휴지통 탭으로 이동
2. 1주일 이상 전의 할일 선택
3. 영구 삭제 버튼 클릭
4. "정말 삭제하겠습니까?" 확인 후 확인

**결과**: 할일이 데이터베이스에서 완전히 제거됨 (기능: FR-011)

---

## 3. 예외 상황

### 완료 표시 실수

**상황**: 완료 체크를 실수로 했는데, 사실 아직 진행 중

**해결**: 완료 목록에서 해당 항목을 다시 클릭하여 체크 해제 (기능: FR-009)

---

### 날짜 입력 오류

**상황**: 시작일(2025-01-05)이 종료일(2025-01-01)보다 뒤에 있음

**결과**: "시작일은 종료일 이전이거나 같아야 합니다" 에러 메시지 표시
사용자가 올바른 날짜 입력 후 다시 시도 (검증: BR-001)

---

### 다른 사용자의 할일 접근 시도

**상황**: 사용자 A가 사용자 B의 할일을 보려고 시도

**결과**: 접근 권한 없음 - "권한이 없습니다" 메시지 표시 (정책: BR-003)
